# Requirements: Toddler Companion

**Defined:** 2026-05-26
**Core Value:** Help a parent turn a few minutes of screen time into calm, guided language and cognition play that connects back to the real world.

## v1 Requirements

### Mobile Shell

- [ ] **SHELL-01**: Parent can open the app on a phone browser and see a touch-friendly home screen.
- [ ] **SHELL-02**: Child-facing controls use large tap targets suitable for toddler-guided tapping with parent support.
- [ ] **SHELL-03**: Parent can start an activity in one or two taps without account setup.
- [ ] **SHELL-04**: App layout remains usable on common mobile viewport sizes.

### Content

- [ ] **CONT-01**: Parent and child can explore a small real-photo vocabulary pack with everyday objects.
- [ ] **CONT-02**: Parent and child can explore colors using real-photo examples.
- [ ] **CONT-03**: Parent and child can explore simple shapes using real-photo or real-world examples.
- [ ] **CONT-04**: Parent and child can explore small numbers from 1 to 5.
- [ ] **CONT-05**: Parent and child can explore simple expressions such as "I want," "thank you," "no," "hug," and "again."
- [ ] **CONT-06**: Each learning item includes parent guidance that suggests what to say or ask.

### Activities

- [ ] **ACT-01**: Child can complete a picture-selection prompt by tapping the matching real photo.
- [ ] **ACT-02**: Child can complete a find-it prompt for a color, shape, object, or number.
- [ ] **ACT-03**: Parent can use a repeat/respond prompt that encourages the child to say or gesture a simple word or expression.
- [ ] **ACT-04**: Activity feedback is calm and non-punitive whether the child taps correctly, taps differently, or exits.
- [ ] **ACT-05**: Activities can be repeated without creating streaks, points, or reward loops.

### Healthy Use

- [ ] **HEALTH-01**: Child or parent can exit an activity at any time.
- [ ] **HEALTH-02**: App starts a gentle wind-down after about 5 minutes of session activity.
- [ ] **HEALTH-03**: Wind-down suggests a real-world parent-child action, such as finding an object nearby or giving a hug.
- [ ] **HEALTH-04**: App does not use points, streaks, leaderboards, random rewards, endless levels, or infinite feeds.
- [ ] **HEALTH-05**: App avoids fast-paced animations, flashing effects, loud reward sounds, and dense multi-prompt screens.

### Progress

- [ ] **PROG-01**: Parent can see recently explored items or activities.
- [ ] **PROG-02**: Parent can see simple exposure progress grouped by vocabulary, colors, shapes, numbers, and expressions.
- [ ] **PROG-03**: Progress language avoids ranking, failure states, or pressure-based mastery claims.
- [ ] **PROG-04**: Progress data is stored locally in v1 without requiring a child account.

### Privacy

- [ ] **PRIV-01**: App works without collecting the child's name, birthdate, voice recordings, or photos.
- [ ] **PRIV-02**: App does not require login, cloud sync, or third-party analytics in v1.

## v2 Requirements

### Content Expansion

- **CONT-V2-01**: Parent can add custom family photos as learning items.
- **CONT-V2-02**: App includes multiple themed content packs such as kitchen, park, animals, clothing, and bedtime.
- **CONT-V2-03**: App supports bilingual labels or prompts.

### Parent Tools

- **PARENT-V2-01**: Parent can configure session length.
- **PARENT-V2-02**: Parent can mark favorite items or hide items that are not useful.
- **PARENT-V2-03**: Parent can export or back up progress.

### Audio

- **AUDIO-V2-01**: App includes high-quality recorded audio for labels and prompts.
- **AUDIO-V2-02**: Parent can choose whether prompt audio plays automatically.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Native iOS or Android app | Mobile Web validates the experience faster. |
| Accounts and cloud profiles | Not needed for v1 and increases child privacy complexity. |
| Third-party analytics | Avoid collecting child usage data before there is a clear privacy-reviewed need. |
| Points, streaks, rewards, leaderboards, or endless levels | Conflicts with the anti-addiction product principle. |
| Video-heavy content | Higher stimulation and less aligned with parent-guided tapping. |
| Formal curriculum or mastery testing | v1 is short parent-child play, not assessment. |
