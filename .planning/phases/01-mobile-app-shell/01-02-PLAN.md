<threat_model>
Trust boundaries:
- Static asset boundary: all demo images are local repository files.
- UI state boundary: feedback/sheet/open-screen state is in-memory only.

STRIDE register:
- T-01-03 Information disclosure: recording demo clicks as progress would create child usage data before Phase 7. Mitigation: keep record state empty and do not write localStorage/IndexedDB/cookies.
- T-01-04 External content injection: remote image URLs could leak requests or break offline behavior. Mitigation: use only `/demo/apple.png`, `/demo/cup.png`, `/demo/shoe.png`.
- T-01-05 Harmful feedback: harsh wrong/failure copy can create pressure. Mitigation: use warm retry copy and no score/error state.
</threat_model>
