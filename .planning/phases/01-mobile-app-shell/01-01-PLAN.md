<threat_model>
Trust boundaries:
- Browser-only static app boundary: no backend, no remote analytics, no account boundary.
- Local asset boundary: demo assets are served from `public/demo/`, not fetched from third-party URLs.

STRIDE register:
- T-01-01 Information disclosure: accidentally collecting child name, birthdate, photos, voice, analytics, or profile data. Mitigation: do not add forms, storage writes, analytics scripts, microphone/camera permissions, auth, or network data submission in Phase 1.
- T-01-02 Spoofing/identity: creating fake account/session concepts that imply user identity. Mitigation: no login/session UI or state.
- T-01-03 Tampering: future code mutates UI-SPEC tokens ad hoc. Mitigation: centralize tokens in `src/styles/tokens.css` and read UI-SPEC before edits.
</threat_model>
