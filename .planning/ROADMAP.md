**Granularity:** Fine
**Project Mode:** Vertical MVP
**Core Value:** Help a parent turn a few minutes of screen time into calm, guided language and cognition play that connects back to the real world.

## Phases

### Phase 1: Mobile App Shell

**Goal:** Create a phone-friendly app foundation that opens quickly, needs no account, and is safe for parent-child use.
**Mode:** mvp
**UI hint:** yes

**Requirements:** SHELL-01, SHELL-02, SHELL-03, SHELL-04, PRIV-01, PRIV-02

**Success Criteria**:
1. Parent can open the app in a phone browser and reach the home screen.
2. The home screen uses large, stable tap targets suitable for toddler-guided use.
3. Parent can start a placeholder activity in one or two taps.
4. No account, child profile, analytics, child name, birthdate, photo upload, or recording is required.
5. Layout is usable on common mobile viewport sizes.

### Phase 2: Real-Photo Content Pack

**Goal:** Create the first structured content pack using real photos and parent guidance.
**Mode:** mvp
**UI hint:** yes

**Requirements:** CONT-01, CONT-02, CONT-03, CONT-04, CONT-05, CONT-06

**Success Criteria**:
1. App can load a local content schema for vocabulary, colors, shapes, numbers, and expressions.
2. Each item includes a real-photo asset or real-world visual example.
3. Each item includes child-facing prompt text and parent guidance.
4. The content pack is small enough for v1 but covers every required category.

### Phase 3: Picture Selection Activity

**Goal:** Deliver the first complete toddler tap activity using picture-selection prompts.
**Mode:** mvp
**UI hint:** yes

**Requirements:** ACT-01, ACT-04

**Success Criteria**:
1. Child can hear or see a prompt such as "Where is the apple?" and tap the matching photo.
2. Correct, incorrect, and exploratory taps receive calm, non-punitive feedback.
3. Parent guidance appears without crowding the child-facing screen.
4. Activity can run with multiple items from the content pack.

### Phase 4: Find-It Activity

**Goal:** Add find-it prompts for colors, shapes, objects, and small numbers.
**Mode:** mvp
**UI hint:** yes

**Requirements:** ACT-02

**Success Criteria**:
1. Child can find an item by color, shape, object name, or number.
2. The activity uses the same calm feedback pattern as picture selection.
3. Prompts remain simple and age-appropriate for a 2-year-old with parent guidance.
4. Find-it activity works across at least three content categories.

### Phase 5: Repeat and Respond Activity

**Goal:** Add simple language prompts that invite the child to repeat, gesture, or respond with parent support.
**Mode:** mvp
**UI hint:** yes

**Requirements:** ACT-03

**Success Criteria**:
1. Parent can start a repeat/respond prompt for vocabulary or simple expressions.
2. Prompt screen gives the parent a short phrase to say and an optional follow-up.
3. The activity can be completed without voice recording or automated speech judgment.
4. The flow supports "try again," "next," and "done" without reward pressure.

### Phase 6: Healthy Session Design

**Goal:** Make healthy use a first-class part of the product.
**Mode:** mvp
**UI hint:** yes

**Requirements:** HEALTH-01, HEALTH-02, HEALTH-03, HEALTH-04, HEALTH-05, ACT-05

**Success Criteria**:
1. Parent or child can exit an activity at any time.
2. A gentle wind-down appears after about 5 minutes of session activity.
3. Wind-down suggests at least one real-world parent-child action.
4. The app contains no points, streaks, random rewards, leaderboards, endless levels, or infinite feeds.
5. Motion, sound, feedback, and screen density stay calm and low-stimulation.
6. Repeating an activity is possible without creating a reward loop.

### Phase 7: Local Progress

**Goal:** Give the parent useful progress visibility without turning the child experience into assessment.
**Mode:** mvp
**UI hint:** yes

**Requirements:** PROG-01, PROG-02, PROG-03, PROG-04

**Success Criteria**:
1. Parent can see recently explored items or activities.
2. Parent can see exposure progress by vocabulary, colors, shapes, numbers, and expressions.
3. Progress copy uses low-pressure language and avoids ranking or failure states.
4. Progress is stored locally and works without an account or cloud sync.

### Phase 8: Integration and Launch Polish

**Goal:** Verify the complete v1 loop and polish the app for real family use.
**Mode:** mvp
**UI hint:** yes

**Requirements:** All v1 requirements

**Success Criteria**:
1. Parent can open the app, start an activity, guide the child, wind down, and review progress.
2. Every v1 requirement is manually or automatically verified.
3. The experience is usable on representative mobile viewport sizes.
4. Content, feedback, and session ending align with the anti-addiction principles.
5. The app is ready for a first real-world trial with the user's child.
